# issue-34 — inline blob:-sourced images in the recorder — Implementation Plan

> **For Claude:** Execute this plan via the ctw build phase. REQUIRED: follow `steps/build.md`.

**Goal:** Make `blob:`-sourced (and cross-origin) `<img>` bytes survive into the
self-contained export by passing `inlineImages: true` to rrweb's `record()`, so a
recorded page's images render offline in a clean tab instead of showing broken.

**Architecture:** One flag at the single place image bytes enter the pipeline
(`src/rrweb-recorder.js`'s `record({...})` literal). rrweb draws each already-loaded
`<img>` onto an offscreen canvas and stores the pixels as an `rr_dataURL` attribute;
everything downstream (relay → background → storage → report-builder → Replayer) is
already agnostic to whether `rr_dataURL` is set. No pipeline change, no new branch, no
egress. The flag is compiled into `dist/rrweb-recorder.js` by `node build.mjs`.

**Tech Stack:** rrweb 2.0.1 (`inlineImages` option, default `dataURLOptions` = PNG),
esbuild bundling via `build.mjs`, `bun test` (unit), Playwright (e2e, real unpacked
extension headless).

**AC map** (`docs/tickets/issue-34.md` — immutable):
- **AC1** — record a `blob:`-image page, export, open in a clean/offline tab → image
  renders, no dead `blob:` sourcing it. Contributed by **Task 1** (the fix) and proven
  by **Task 3** (e2e, asserted on the rendered replay image).
- **AC2** — disconfirming: the same page recorded *without* the fix still shows the
  broken image, proving the flag is what closes it. Contributed by **Task 2** (unit
  guard: the durable, in-CI causal lever) and **Task 3** (e2e comment + one-time manual
  confirmation). CI runs `npm run build` before tests, so removing the flag from source
  rebuilds `dist/` without it and turns both the unit and e2e assertions red.

**Load-bearing design fact** (from `issue-34-design.md` §Architecture): rrweb *retains*
the original `attributes.src="blob:..."` and adds `rrweb-original-src` on replay; the
CSP meta also literally contains `blob:`. A raw "export string contains no `blob:`"
scan therefore false-fails even with the fix. **AC1 must be asserted on the rendered
replay image** (effective `src` / `naturalWidth`), never on the raw export string.

      inlineImages: true,
      emit(event) {
        buffer.push(event);
        if (!flushTimer) {
          flushTimer = setTimeout(() => {
            flushTimer = null;
            flush();

        }
      },
    });

**Step 1: Extend `recordSession()` to append a loaded `blob:` `<img>` before `start`**

The full snapshot happens at `record()` start, so the image must exist and be loaded
*before* the `start` action is sent (line 43). Add the setup between `goto` and `start`,
and return the captured blob URL.

```js
// A 1x1 PNG as a data: URI — the blob's byte source is irrelevant to the bug;
// only that the <img> is sourced from a blob: object URL (like Atlassian Media).
const TINY_PNG =
  "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAQAAAC1HAwCAAAAC0lEQVR42mP8z8BQDwAEhQGAhKmMIQAAAABJRU5ErkJggg==";

async function recordSession({ injectStyle = false, blobImage = false } = {}) {
  const fixture = await context.newPage();
  await fixture.goto(fixtureServer.url, { waitUntil: "load" });

  let blobUrl = null;
  if (blobImage) {
    // Faithful repro: fetch a data: PNG -> blob -> object URL -> <img>, and wait
    // until it has actually decoded (naturalWidth > 0) so the recorder hits
    // rrweb's synchronous inline path deterministically (no load-event race).
    blobUrl = await fixture.evaluate(async (src) => {
      const blob = await (await fetch(src)).blob();
      const url = URL.createObjectURL(blob);
      const img = document.createElement("img");
      img.id = "blobimg";
      img.src = url;
      document.body.appendChild(img);
      if (!(img.complete && img.naturalWidth > 0)) {
        await new Promise((res) => { img.onload = res; img.onerror = res; });
      }
      return url;
    }, TINY_PNG);
  }

  const popup = await openPopup(context, extensionId);
  const tabId = await tabIdOf(popup, fixtureServer.url);
  const started = await sendAction(popup, { action: "start", tabId });
  expect(started.ok).toBe(true);

  await fixture.bringToFront();
  if (injectStyle) await fixture.locator("#injectStyle").click();
  await fixture.locator("#name").fill("Ada Lovelace");
  for (let i = 0; i < 3; i++) await fixture.locator("#inc").click();
  await fixture.locator("#fetchBtn").click();
  await expect(fixture.locator("#counter")).toHaveText("3");
  await fixture.waitForTimeout(1200);

  await sendAction(popup, { action: "screenshot" });
  const viewer = await stopAndOpenViewer(context, popup);
  await fixture.close();
  return { viewer, popup, blobUrl };
}
```

> Note: existing callers pass no `blobImage`, so they are unaffected (default `false`).

**Step 2: Add the offline acceptance test (AC1)**

Model it on `:155-181`. Download the export, kill the fixture server (fully offline),
open the export from disk, and assert on the **rendered replay image** — not the raw
string.

```js
test("exported report inlines blob:-sourced images so they render offline", async () => {
  // AC1: a blob: <img> (Atlassian Media pattern) must survive into the export and
  // render in a clean/offline tab. Asserted on the RENDERED replay image, not the
  // raw export string: rrweb retains the original src="blob:..." attribute (adds
  // rrweb-original-src) and the CSP meta literally contains "blob:", so a raw-string
  // scan false-fails even with the fix.
  //
  // AC2 disconfirming input: remove `inlineImages: true` from src/rrweb-recorder.js
  // and rebuild (`node build.mjs`) — the replay <img> keeps its dead blob: src,
  // naturalWidth === 0, image broken. CI rebuilds before tests, so this is the same
  // lever the unit guard pulls; confirm once by hand (mirrors the waveform test's
  // comment-plus-one-run convention at e2e/extension.spec.mjs:130-133).
  const { viewer, popup, blobUrl } = await recordSession({ blobImage: true });

  const [download] = await Promise.all([
    viewer.waitForEvent("download"),
    viewer.locator("#download").click(),
  ]);
  const file = path.join(mkdtempSync(path.join(tmpdir(), "openjam-e2e-")), "export.html");
  await download.saveAs(file);

  await fixtureServer.close(); // fully offline — a dead blob: src cannot resolve
  const offline = await context.newPage();
  await offline.goto("file://" + file, { waitUntil: "domcontentloaded" });

  const frame = offline.frameLocator(".replayer-wrapper iframe");
  const img = frame.locator("#blobimg");
  await img.waitFor();
  const info = await img.evaluate((el) => ({
    w: el.naturalWidth,
    src: el.currentSrc || el.src,
    orig: el.getAttribute("rrweb-original-src"),
  }));
  expect(info.w).toBeGreaterThan(0);            // it renders — no "Failed to load image"
  expect(info.src.startsWith("data:")).toBe(true); // sourced from inlined bytes, not blob:
  expect(info.orig).toBe(blobUrl);              // Replayer swapped rr_dataURL in for the blob url

  await offline.close();
  await viewer.close();
  await popup.close();
  fixtureServer = await serveFixture(); // restore for later serial tests
});
```

> The `<img>` lives in the initial full snapshot, so it appears in the replayed DOM on
> mount — no Play needed. If `img.waitFor()` is flaky, fall back to the `:172-175`
> pattern (set 8x, Play, wait for `#counter` = 3) before querying the image.

## Open items for the orchestrator

- **AC1 wording** ("no `blob:` `src` left in the serialised DOM") is interpreted, per
  the approved design, as "the image no longer *renders from* a dead `blob:`" — asserted
  on the rendered replay image's effective `src`/`naturalWidth`. rrweb deliberately
  retains the raw `blob:` attribute; stripping it would need a post-record transform
  (larger scope). This reading is baked into the plan; flag if the ticket wants the raw
  attribute stripped instead.
- **`dataURLOptions`** ships as default PNG (`unlimitedStorage` lifts the quota). The
  JPEG size knob is documented in the Task 1 comment as a reversible fast-follow, not
  shipped now (YAGNI).
